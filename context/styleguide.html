Style guide

The visual and editorial system for the personal site. One accent, warm near-black on cream, a single serif doing the work. When in doubt, remove something. This page is written in the system it describes.

01 — Typeface

Newsreader, throughout

A single serif for headings and body. It reads like an essay, which is the point: the site is mostly about explaining and writing. Weights are limited to regular (400) and medium (500) — never bolder. Italics are reserved for section labels and the occasional aside. A monospace stack (ui-monospace) is the only other face, used for hex values and code, never for prose.

Never pure black on pure white — the warmth of the cream and the softened ink are most of why the site feels calm. The brick-red accent appears only on links and nav. If something needs a second color to be understood, the layout is doing too little work.

04 — Spacing & measure

Air is the layout

Content is centered in a single column capped at 34rem (~560px), which holds roughly 50–55 characters per line — the width that makes sparse text read as composed rather than empty. Hold this measure on every page.

section gap · 56pxlabel → content · 16pxparagraph · 14pxrow padding · 7pxpage margin · 64px top

Vertical rhythm is generous and regular. Sections are separated by a top hairline and the 56px gap; nothing is boxed or shadowed except, on this reference page, the demo frames.

05 — Navigation

Two placements

The homepage puts nav beneath the intro statement, so the name and the first-person line are read before anything else. Every interior page puts the same nav in a top bar, with the name as a small wordmark linking home.

Order is fixed: About · Projects · Teaching · SciComm · CV. Links are plain accent text, no buttons, no underline until hover — regular weight, never bold; the accent color alone carries the emphasis. On mobile the top-bar nav wraps to a second line rather than collapsing into a menu. On the homepage only, entries are joined with a centered dot (· in var(--faint)) so the row reads as one feature line, preceded by a hairline divider under the intro paragraph; the interior top-bar nav stays plain-gapped, no dots.

06 — Recurring elements

The few pieces the site is built from

Intro statement

Two to three sentences, first person, present tense. The only place the site speaks directly. It states field, angle, and what you make — no title stacking, no buzzwords.

IPA nameplate

Ricky Rojas
/ˈrɪ.ki ˈro.xas/

Sits directly beneath the h1 on the homepage only — nowhere else. Uses the serif at 17px, weight 400, color var(--faint), letter-spacing .01em. Not italic; IPA is a phonetic system, not emphasis. The transcription uses slashes (phonemic), not square brackets (phonetic). Do not add a label or explanation — the notation is self-evident to anyone who reads it, and pleasantly cryptic to anyone who doesn't.

08 — Voice

How the words sound

  • First person, plain. “I build tools and teach,” not “Ricky is a researcher specializing in…”.
  • Specific over clever. Name the actual field and the actual thing you made.
  • Sentence case everywhere — headings, labels, links.
  • Let each line do one job. A label labels; a description describes; nothing does double duty.
  • Cut one thing. Before a section ships, remove the least necessary sentence.
